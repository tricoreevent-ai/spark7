## 📝 Manual Git Commands

### Basic Operations
```bash
# Show status
git status

# View changes
git diff
git diff --cached

# Stage files
git add .
git add file.txt
git add src/

# Commit
git commit -m "Your message"

# Push
git push origin main

# Pull
git pull origin main
```

### Branching
```bash
# Create branch
git checkout -b feature/name

# Switch branch
git checkout main
git switch feature/name

# List branches
git branch
git branch -r

# Delete branch
git branch -d feature/name
```

### History
```bash
# View log
git log
git log --oneline
git log --oneline -10
git log --graph --oneline --all

# View specific commit
git show commit-hash
```

### Undoing Changes
```bash
# Discard local changes
git checkout -- file.txt

# Unstage files
git reset HEAD file.txt

# Amend last commit
git commit --amend -m "New message"

# Revert commit
git revert commit-hash
```

## 🔧 Setup (One-time)

### Configure Git
```bash
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

### Set Remote
```bash
git remote set-url origin https://github.com/tricoreevent-ai/spark7
# or
git remote add origin https://github.com/tricoreevent-ai/spark7
```

### GitHub Authentication
```bash
# Option 1 - SSH
ssh-keygen -t ed25519 -C "your@email.com"
# Add public key to GitHub Settings > SSH

# Option 2 - Personal Access Token
# Use as password when pushing
```

### Check Setup
```bash
git-setup.bat
```

## ⚠️ Common Issues

### Authentication Failed
```bash
# Update credentials in Windows Credential Manager
# Or regenerate GitHub token
```

### Changes Not Appearing
```bash
git status                    # Check status
git add -A                    # Stage all
git commit -m "message"       # Commit
git push origin main          # Push
```

### Wrong Commit Info
```bash
git config user.name          # Check name
git config user.email         # Check email
git config --global user.name "name"
git config --global user.email "email"
```

### Can't Push
```bash
git pull origin main          # Pull first
git push origin main          # Try push again
```

## 📊 Typical Commit Sequence

### Per Feature
```
1. Create branch
   git checkout -b feature/xyz

2. Make changes
   [edit files]

3. Commit regularly
   commit-advanced.bat (multiple times)

4. Push changes
   git push origin feature/xyz

5. Create Pull Request on GitHub
```

### Per Session
```
1. Start
   git pull origin main

2. Work
   [edit multiple files]

3. Commit batches
   git add file1 file2
   git commit -m "feat: Group related changes"
   
4. Repeat until done
   [edit more files]

5. Push all
   git push origin main
```

## 🚨 Emergency Commands

### Undo last commit (not pushed)
```bash
git reset --soft HEAD~1
```

### Discard all changes
```bash
git checkout -- .
```

### View lost commits
```bash
git reflog
```

### Cancel ongoing merge
```bash
git merge --abort
```

## 📌 Pro Tips

1. **Commit often** - Multiple small commits are better than one large commit
2. **Clear messages** - Write descriptive commit messages
3. **Use types** - Use feat/fix/update for better organization
4. **Pull before push** - Always sync before pushing
5. **Review changes** - Check what you're committing with `git diff`
6. **Backup locally** - Keep local commits even if push fails
7. **Branch for features** - Create branches for new features
8. **Keep history clean** - Squash commits before merging if needed
